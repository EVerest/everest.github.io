1. EVerest framework¶

See our Quick Start Guide for a fast overview and first steps to create new modules: A Real Quick Guide To EVerest

EVerest uses the principle of loose coupling of modules that allow configuration during runtime.

1.1. Modules¶

Modules consist of one or more interfaces. Each module is defined by its manifest.json file:

JSON KEY

description

value type

description

useful description of what the module does

string

provides

interfaces provided by the module, can be required by other modules

{<interface id1>: {interface, description, [config]}, <interface id2>: {interface, description, [config]}, … }

requires

interfaces required by the module, needs to be provided by other module(s)

{<interface id1>: {interface}, <interface id2>: {interface},…}

metadata

metadata

{license: <string>, authors: <array>}

An example module can be found here.
