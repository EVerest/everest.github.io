4 /*
5 The IECStateMachine class provides an adapter between the board support package driver (in a seperate module) and the
6 high level state machine in Charger.cpp.
7
8 Typically the CP signal generation/reading and control of contactors/RCD etc is handled by a dedicated MCU. This MCU
9 and/or the HW is responsible for the basic electrical safety of the system (such as safely shut down in case of RCD
10 trigger or Linux crashing). The BSP driver is just a simple HW abstraction layer that translates the commands for
11 setting PWM duty cycle/allow contactors on as well as the CP signal readings/error conditions into the everest world.
12 It should not need to implement any logic or understanding of the IEC61851-1 or any higher protocol.
13
14 This IECStateMachine is the low level state machine translating the IEC61851-1 CP states ABCDEF into more useful
15 abstract events such as "CarPluggedIn/CarRequestedPower" etc. These events drive the high level state machine in
16 Charger.cpp which handles the actual charging session and coordinates IEC/ISO/SLAC.
17
18 */
