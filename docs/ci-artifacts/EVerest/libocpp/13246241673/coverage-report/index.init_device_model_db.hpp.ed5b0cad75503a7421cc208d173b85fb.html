9 /// @class ocpp::v201::InitDeviceModelDb
10 /// @brief Class to initialize the device model db using the component config files
11 ///
12 /// This class will read the device model config files and put them in the device model database.
13 ///
14 /// If the database already exists and there are some changes on components, this class will make the changes in the
15 /// database accordingly.
16 ///
17 /// It will also re-apply the component config. Config items will only be replaced if they are changed and the value in
18 /// the database is not set by an external source, like the CSMS.
19 ///
20 /// The data from the component config json files or database are read into some structs. Some structs could be reused
21 /// from the DeviceModelStorage class, but some members are missing there and and to prevent too many database reads,
22 /// some structs are 'redefined' in this class with the proper members.
23 ///
24 /// Since the DeviceModel class creates a map based on the device model database in the constructor, this class should
25 /// first be finished with the initialization before creating the DeviceModel class.
26 ///
27 /// The config values are updated every startup as well, as long as the initial / default values are set in the
28 /// database. If the value is set by the user or csms or some other process, the value will not be overwritten.
29 ///
30 /// Almost every function throws exceptions, because this class should be used only when initializing the chargepoint
31 /// and the database must be correct before starting the application.
32 ///
